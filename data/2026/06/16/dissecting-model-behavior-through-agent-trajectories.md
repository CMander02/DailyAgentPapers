# Dissecting model behavior through agent trajectories

## 原始摘要

AI agent performance is not just a modeling problem, it is fundamentally a systems problem. The advanced capabilities of models are realized through agent harnesses. Therefore, a gap between model assumptions and harness behavior can easily prevent the model's full capabilities from translating into agent performance. We formalize this as the `intent-execution' gap: the mismatch between what the model intends and what the harness executes, and vice versa. We argue that minimizing this intent-execution gap is as important as other aspects of harness design such as tools and execution loops. To illustrate the impact of this harness-model alignment, we develop a simple and customizable harness called `Simple Strands Agent' (SSA). SSA aims to find the bulk of common patterns which generalize across different model families (such as Claude, Gemini, GPT, Grok, Qwen), as well as a small number of model-specific preferences. We make two contributions: (i) we $\textbf{reproduce or improve on the pass@1}$ performance reported by diverse model-provider families on popular agentic benchmarks (SWE-Pro, SWE-Verified and Terminal-Bench-2), and (ii) building on an $\textbf{analysis of 138k trajectories generated by SSA}$, we look beyond the $\texttt{pass@1}$ numbers which tend to be relatively even across frontier models. By representing agent trajectories in code state-spaces, we observe model-level differences in problem-solving behavior. Finer-grained metrics such as edit frequency, testing activity, and phase-transitions reveal how individual models allocate effort across different stages of autonomous problem solving.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决AI Agent系统中模型能力与执行表现之间存在显著差距的问题。研究背景是，当前AI Agent的性能不仅取决于基础模型的原始能力，更依赖于“harness”（即连接模型与外部环境的软件层）的设计。现有方法往往针对特定模型和基准进行定制化调优（如专用提示词、工具模式），但这些增益是脆弱的，模型升级后容易失效，且忽略了模型中意图与执行环境之间的根本性不匹配。论文将这种双向不匹配形式化为“意图-执行鸿沟”：一方面，模型意图执行的操作可能因解析错误、工具命名冲突等原因而被harness错误执行；另一方面，环境执行结果可能被截断或误传，导致模型基于错误前提推理。这一鸿沟会导致模型在看似正确的推理路径上反复失败，甚至放弃有效方案。核心问题是，如何在不依赖任务特定调优的前提下，设计一种通用的、跨模型家族的harness方案，来最小化意图-执行鸿沟，从而充分释放模型潜力。论文通过提出Simple Strands Agent（SSA）和基于138k轨迹的细粒度分析，证明缩小这一鸿沟足以匹配甚至超越官方基准性能，并揭示模型间不同的行为模式。

### Q2: 有哪些相关研究？

本文的相关工作主要分为三类：

**方法类与研究**：现有工作通常针对特定模型（如Claude、GPT）设计定制化工具架构、提示词或执行图谱，以缩小模型意图与执行结果之间的差距。本文的“Simple Strands Agent”（SSA）与此不同，它不追求模型特定优化，而是提出“意图-执行差距”这一通用概念，并设计跨模型家族（Claude、Gemini、GPT、Grok、Qwen）均有效的接口原则，仅对少数家族特有偏好进行适配，从而避免了对特定模型的过拟合。

**评测与基准类研究**：现有基准（如SWE-Bench-Verified、SWE-Bench-Pro、Terminal-Bench-2）通常以pass@1作为核心指标，掩盖了不同模型在问题解决行为上的差异。本文不仅在这些基准上复现或超越了官方报告的性能，还首次基于138k轨迹进行了细粒度分析，通过编辑频率、测试活动和阶段转换等指标揭示模型特有的行为特征，且发现这些特征在pass@1数值相近的模型中依然稳定存在。

**系统与基础设施类研究**：本文指出了一个关键的系统问题——git历史泄露渠道，该问题在公开的SWE-Bench-Pro容器中导致pass@1指标被高估最多6.9%。这一发现与通常仅关注模型能力的现有工作形成区别，强调了智能体性能在本质上是系统问题而非单纯的建模问题。

### Q3: 论文如何解决这个问题？

该论文通过提出**Simple Strands Agent (SSA)**框架，从系统层面解决模型能力与智能体表现之间的“意图-执行差距”。核心方法围绕三个关键技术展开：

**1. 反馈丰富的工具边界**：SSA设计小型工具集（bash、文件编辑/查看），但通过结构化执行反馈增强边界质量。每个shell调用返回完整元数据（命令、状态、退出码、输出、错误文本），超时时保留部分输出。编辑操作采用邻行锚定机制，拒接局部匹配和重复匹配并给出消歧反馈，成功编辑直接返回统一差异（diff），减少模型验证负担。

**2. 推理引导与工具调用平衡**：针对不同模型家族采用定制化推理策略。对Claude系列使用量化引导（如“执行50+次工具调用”）打破长链推理，而Gemini和Grok系列则采用灵活指令（如“尽可能使用工具”），避免其字面理解导致空洞调用。核心原则是激励模型用足够推理决定下一步行动后，优先通过环境验证假设而非继续内部思考。

**3. 解决方案距离度量**：创新地将智能体轨迹映射到代码状态空间，定义文本级差异投影φᵢ(x)和召回偏差Dᵢ(t)=1−|φᵢ(x(t))∩φᵢ(x*)|/|φᵢ(x*)|，量化当前状态与最近正确修复的接近程度。通过138k条轨迹分析，发现编辑频率、测试活动和阶段转换等细粒度指标能揭示模型在自主问题解决不同阶段的努力分配差异，弥补pass@1指标无法反映轨迹质量（如回溯、效率）的不足。

SSA整体架构由任务输入、提示生成、模型适配器、核心事件循环（交替模型调用与工具执行）、Docker环境、钩子（上下文管理、内容过滤、轨迹记录）组成，遵循“满足模型偏好”的设计原则（如GPT的apply_patch格式适配、Grok的原子化工具拆分），最终在SWE-Pro等基准上复现或超越原有pass@1性能。

### Q4: 论文做了哪些实验？

论文在三个Agent基准上评估了21个模型：SWE-Bench-Verified (n=500)、SWE-Bench-Pro (n=731)和Terminal-Bench-2 (n=89)，涵盖Claude、GPT、Gemini、Grok、Qwen五个模型族。使用统一框架SSA，仅进行极少的家族适配器调整（如提示推理提示和工具表面选择）。每个模型运行5次，报告平均pass@1和95%置信区间。主要结果：在SWE-Bench-Verified上，pass@1范围51.4%-80.8%，Opus 4.6和Gemini 3.1 Pro并列第一（80.80%）；在SWE-Bench-Pro上，pass@1范围27.4%-59.9%，GPT-5.4领先（59.90%）；在Terminal-Bench-2上，受约束设置下GPT-5.3（75.05%）和Opus 4.6（73.93%）领先，无约束设置下Opus 4.6最高（83.14%）。SSA复现或超过了所有模型的官方公布成绩（仅一个例外）。此外，分析了138k条轨迹，通过将代码状态空间中的轨迹表示为编辑频率、测试活动和阶段转换等细粒度指标，揭示了不同模型族在问题解决行为上的差异，如各模型在探索/定位/实现/验证阶段的精力分配。

### Q5: 有什么可以进一步探索的点？

论文提出的Solution Distance指标虽然创新性地度量了轨迹质量，但存在几处可深化的方向。首先，当前文本级别的代码空间忽略了语义等价性，一个经过正确重构或变量重命名的状态虽在语义上正确，却被判定为高dist离，未来可引入语义等价类或AST结构相似度来改进。其次，D_i(t)的近似依赖有限参考集，当真实解空间存在多个不相交的编辑策略时，min操作可能无法反映真实进度，可探索多原型距离或多模态解空间的度量。此外，论文主要关注单实例内轨迹，缺乏跨任务迁移行为分析。未来可以研究模型在类似任务间是否表现出一致的“行为指纹”，如调试策略或回滚模式，从而构建模型能力的行为画像。最后，结合强化学习，可尝试将Solution Distance作为奖励信号，直接优化模型在轨迹中减少回溯、提升编辑效率，这可能比单纯优化pass@1更利于训练稳健的agent。

### Q6: 总结一下论文的主要内容

这篇论文提出并形式化了AI Agent系统中“意图-执行差距”的概念，即模型意图与执行框架（Harness）实际执行之间的双向不匹配。为最小化这一差距，作者开发了一个简单可定制的执行框架SSA（Simple Strands Agent）。SSA在SWE-Bench等主流Agent基准测试中，对Claude、Gemini、GPT等多个模型家族复现或超越了官方pass@1性能。更关键的贡献是，通过分析SSA生成的13.8万条轨迹，作者引入了一种基于代码状态空间的解决方案距离度量，从而超越了简单的二元胜负评判。细粒度的行为指标（如编辑频率、测试活动阶段转换）揭示了不同模型家族在问题解决过程中的稳定行为特征，这些差异在传统聚合指标中不可见。该研究强调了模型-执行框架对齐的重要性，并为深入理解和诊断Agent行为提供了新方法。
