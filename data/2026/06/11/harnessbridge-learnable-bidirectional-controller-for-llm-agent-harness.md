# HarnessBridge: Learnable Bidirectional Controller for LLM Agent Harness

## 原始摘要

Large language models are increasingly deployed as agents for long-horizon tasks, yet their performance is shaped not only by model capability and environment design, but also by the harness that mediates agent--environment interaction. Existing harnesses are largely manually engineered, making them difficult to scale as trajectories grow longer and interactions become more complex. In this work, we ask whether harness can be generated by a learnable plug-in module that can be trained in an end-to-end fashion. We introduce HarnessBridge, a lightweight learnable harness controller that parameterizes the agent--environment interface as a bidirectional projection. HarnessBridge learns two bidirectional projections: observation projection, which distills raw trajectories into compact, decision-relevant states, and action projection, which converts proposed actions into executable transitions or trajectory-grounded rejections. We train HarnessBridge on a harness supervision dataset via unified instruction tuning. On Terminal-Bench~2.0 and SWE-bench Verified, HarnessBridge matches or surpasses strong specialized harnesses while substantially reducing token usage and trajectory length, and generalizes from smaller generators to larger commercial models.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文试图解决LLM Agent系统中“控制程序”（harness）的工程化瓶颈问题。研究背景是，随着LLM被广泛部署为执行长周期任务的智能体（如软件工程、网页导航等），其性能不仅取决于模型能力和环境设计，还依赖于控制程序这一中介接口。现有方法大多依赖手动设计的控制程序，存在两个关键不足：1）在环境到Agent方向上，轨迹中积累了冗余上下文、过期错误、低价值细节，导致token成本激增并掩盖决策关键信息；2）在Agent到环境方向上，模型输出可能包含无效重复动作、空循环或格式错误命令，消耗环境步数却无进展。这些手动设计难以随任务轨迹增长和交互复杂化而扩展。

本文要解决的核心问题是：能否将控制程序转化为一个可端到端学习的可插拔模块？具体而言，论文提出HarnessBridge，通过学习双向投影策略来参数化Agent-环境接口——观察投影将原始轨迹压缩为决策相关状态，动作投影将提议动作转化为可执行转换或轨迹驱动的拒绝反馈。通过统一指令微调在控制监督数据集上训练，该方法在长周期基准测试中匹配或超越专用控制程序性能，同时大幅降低token消耗和轨迹长度，并展现从较小生成器到大型商业模型的良好泛化能力。

### Q2: 有哪些相关研究？

相关工作主要分为三类。**第一类是手动设计的启发式方法类**，包括轨迹摘要、基于检索的记忆、上下文压缩、重试规则和工具调用验证等。这些方法通常被实现为静态规则或单独工程化的模块，如WebArena、SWE-bench中的工具使用框架。本文与它们的核心区别在于：这些方法不学习运行时交互策略，而本文提出的HarnessBridge通过可学习的双向投影器，动态决定每个步骤中观察和动作的传递。**第二类是自动化框架优化类**，如Meta-Harness、自动工作流优化、提示选择、脚手架搜索等。它们将整个外部框架（scaffold）作为优化目标，而本文则聚焦于代理-环境接口本身，学习一个持续的、端到端的运行时策略，而非优化外部脚手架。**第三类是应用评测类**，包括终端操作（Terminal-Bench）、软件工程（SWE-bench）和网页导航等基准。本文在这些基准上验证了方法的效果，与现有专用工具相比，在匹配或超越性能的同时大幅减少了token使用和轨迹长度。

### Q3: 论文如何解决这个问题？

HarnessBridge将传统的手工设计的agent与环境交互接口转化为一个可学习的双向控制模块。核心方法是将接口参数化为两个可学习的投影：观测投影和动作投影。观测投影负责将原始交互历史蒸馏为紧凑且与决策相关的状态，暴露给生成器。它学习对每个历史单元进行Pass、Compress或Drop操作，并生成一个主动状态索引，记录未解决的错误、待满足的约束、已建立的事实等关键信息，避免生成器从长轨迹中重构状态。动作投影则负责将生成器提出的动作转换为可执行的转换或被轨迹证据支撑的拒绝。它预测Pass或Reject，当拒绝时返回基于轨迹的反馈（包括原因、证据和建议），减少无效环境交互步骤。这两个投影通过共享的参数化策略模型P_θ实现，仅通过不同的指令和目标格式区分任务。模型通过指令微调在监督数据集上训练，数据由提示的指令微调模型生成，并经过成功轨迹过滤和质量评估。整个框架保留了原始轨迹作为权威记录，投影不破坏历史，仅影响生成器所见状态和环境所执行动作，实现了轻量级、端到端可训练的接口优化。

### Q4: 论文做了哪些实验？

论文在Terminal-Bench 2.0和SWE-bench Verified两个基准上进行了实验，报告了成功率和平均token消耗量。对比方法包括Terminus 2、Terminus-KIRA、mini-SWE-agent、OpenHands、Qwen-Coder等人工设计harness，以及自动优化的Meta-Harness。使用7个冻结的生成器模型：Qwen3.5-35B-A3B、GLM4.7-Flash（开源，温度0.6）、DeepSeek-V4-Flash/Pro、GPT-5.4-nano/5.4、Claude-Opus-4.7（商业API）。主要结果：在Terminal-Bench 2.0上，HarnessBridge在Qwen3.5-35B-A3B上取得33.7%成功率（比Terminus 2提升11.2%），token消耗减少46.8%；在GLM-4.7-Flash上取得20.2%成功率（提升5.8%），token减少77.5%。在SWE-bench Verified上保持竞争力且token最少（Qwen3.5-35B-A3B：60.2%成功率，1.13M token）。跨模型泛化测试显示，HarnessBridge在GPT-5.4-Nano上成功率从18.0%提升至22.5%，token减少90.7%。消融实验表明，动作投影和观测投影均对成功率有贡献，移除任一模块都会降低性能。

### Q5: 有什么可以进一步探索的点？

**局限性与未来方向：**

当前HarnessBridge主要在terminal环境和固定任务上验证，其双向投影机制对高度动态、多模态环境（如具身智能）的泛化能力尚不明确。未来可探索以下方向：

1. **动态参数化与自适应学习**：当前投影器为固定结构，在极端长轨迹中可能面临梯度消失或决策瓶颈。可引入基于强化学习的元学习框架，使投影器根据任务阶段自动调整压缩策略。

2. **冲突解决与安全约束**：动作投影中的“可执行拒绝”机制较简单，未处理多智能体协作时的冲突。可结合对抗训练或安全RL，设计显式的非兼容性检测模块。

3. **跨环境零样本迁移**：论文仅在两个基准上验证，未来可训练通用投影基底，通过LoRA微调适配不同环境接口，减少重复训练成本。

4. **可解释性增强**：当前投影操作如同黑箱，可尝试在观测投影中引入因果干预，可视化哪些轨迹片段被保留/丢弃，提升错误溯因能力。

### Q6: 总结一下论文的主要内容

本文提出 HarnessBridge，一种可学习的轻量级控制器，用于替代传统人工设计的 LLM 智能体与环境交互的“马具”（harness）。核心问题在于，随着任务轨迹变长、交互变复杂，手动设计的马具难以扩展。方法上，HarnessBridge 将智能体-环境接口参数化为双向投影：观测投影将原始轨迹压缩为决策相关的紧凑状态；动作投影将智能体提出的动作转化为可执行的环境转换或基于轨迹的拒绝。通过在 Harness 监督数据集上进行统一指令微调来训练。在 Terminal-Bench 2.0 和 SWE-bench Verified 上的实验表明，HarnessBridge 在匹配甚至超越强专用马具性能的同时，显著降低了 token 使用量和轨迹长度，并能从小模型泛化到大型商业模型。其意义在于提出了可端到端训练的学习型马具，为 LLM 智能体在复杂长程任务中的高效部署提供了新范式。
