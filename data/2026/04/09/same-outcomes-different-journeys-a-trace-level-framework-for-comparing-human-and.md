# Same Outcomes, Different Journeys: A Trace-Level Framework for Comparing Human and GUI-Agent Behavior in Production Search Systems

## 原始摘要

LLM-driven GUI agents are increasingly used in production systems to automate workflows and simulate users for evaluation and optimization. Yet most GUI-agent evaluations emphasize task success and provide limited evidence on whether agents interact in human-like ways. We present a trace-level evaluation framework that compares human and agent behavior across (i) task outcome and effort, (ii) query formulation, and (iii) navigation across interface states. We instantiate the framework in a controlled study in a production audio-streaming search application, where 39 participants and a state-of-the-art GUI agent perform ten multi-hop search tasks. The agent achieves task success comparable to participants and generates broadly aligned queries, but follows systematically different navigation strategies: participants exhibit content-centric, exploratory behavior, while the agent is more search-centric and low-branching. These results show that outcome and query alignment do not imply behavioral alignment, motivating trace-level diagnostics when deploying GUI agents as proxies for users in production search systems.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决当前LLM驱动的GUI智能体在模拟人类用户进行系统评估和优化时，其行为与真实人类行为可能存在系统性差异的问题。研究背景是，随着多模态GUI智能体的快速发展，它们已能直接在真实应用环境中执行复杂的信息搜索任务，并被越来越多地用作人类用户的代理，以进行A/B测试或可用性研究。然而，现有评估方法主要关注智能体的元素定位能力和端到端任务成功率，这种基于结果的评估无法揭示智能体在交互过程中所采用的策略是否与人类相似。

现有方法的不足在于，仅凭任务成功率和查询匹配度等指标，无法全面评估智能体与人类行为的对齐程度。如果智能体通过截然不同的导航策略达成相同目标，那么基于智能体行为轨迹来优化或评估的系统，可能会学习到人类用户并不具备的行为模式，从而导致系统设计出现偏差。

因此，本文要解决的核心问题是：如何超越结果层面的评估，深入分析和比较GUI智能体与人类用户在信息搜索任务中的行为轨迹差异。为此，论文提出了一个追踪层面的评估框架，从任务结果与努力程度、查询表述模式以及跨界面状态的导航行为三个维度进行系统比较。通过在真实的生产级音频流媒体搜索应用中进行受控的人机对比实验，论文揭示了智能体虽然在任务成功率和查询语言上与人类对齐，但其导航策略却更加以搜索为中心、路径分支更少，与人类以内容为中心、更具探索性的行为模式存在显著差异。这证明了结果对齐并不等同于行为对齐，强调了在将智能体作为用户代理部署时，进行追踪层面诊断的必要性。

### Q2: 有哪些相关研究？

本文的相关研究主要可分为三类：方法类、评测类和应用类。

在**方法类**研究中，早期GUI代理依赖于结构化界面表示，而当前基于LLM和多模态感知的视觉代理则能直接处理像素、布局和文本，并通过特定工具链和预训练规划点击、滑动等操作序列。本文延续了这一技术路线，但将代理置于真实生产系统而非模拟环境中运行。

在**评测类**研究中，现有工作如Mind2Web、VisualWebArena等基准测试主要关注任务完成率，虽记录交互轨迹，但缺乏对代理与界面交互策略的深入分析，且多在静态环境中评估。本文提出的轨迹级评估框架则系统比较了人类与代理在任务成果、查询表述和界面状态导航三方面的行为差异，弥补了现有评测对行为对齐关注的不足。

在**应用类**研究中，近期工作探索将LLM代理用作界面评估的可扩展代理，例如基于代理的可用性测试或A/B测试。本文与此类研究动机一致，但进一步强调当代理轨迹用于指导系统设计时，必须确保其交互行为反映真实用户行为。此外，信息检索领域长期使用用户模拟（包括传统基于假设的模拟器和近期LLM生成的搜索轨迹）来研究用户行为，本文则将模拟扩展到生产搜索系统中端到端的GUI交互轨迹对比，重点关注查询表述和状态导航的行为保真度。

### Q3: 论文如何解决这个问题？

论文通过提出并实例化一个**轨迹级别的评估框架**来解决“如何系统比较人类与GUI智能体在生产搜索系统中的行为差异”这一问题。该框架的核心在于超越仅关注任务成功率的传统评估，从三个互补的行为维度进行细粒度比较。

**整体框架与主要模块**：框架包含三个核心组成部分：
1.  **设计多跳信息寻求任务**：创建了10个需要跨搜索结果和实体页面整合信息的复杂搜索任务（如表1所示），分为线性链式和实体桥接两种推理模式，确保任务可完全在应用内完成且具有客观的成功验证标准。
2.  **收集对齐的人类与智能体交互轨迹**：在相同的生产环境（Spotify音频流媒体应用）中，让39名人类参与者和一个先进的GUI智能体（基于AppAgent框架与GPT-5模型）执行相同的任务，并记录其完整的交互日志（包括动作、元素、时间戳和最终播放的曲目）。
3.  **沿三个维度进行行为比较**：这是框架的分析核心，包括：
    *   **任务结果与努力程度**：比较任务成功率、执行动作数和总任务时间。
    *   **查询表述模式**：分析首次查询的相似性（使用字符序列匹配和TF-IDF余弦相似度），并对完整查询集进行分布分析（计算典型性、覆盖率和效率）。
    *   **导航行为**：比较在应用界面状态间的转换模式。

**关键技术细节与创新点**：
1.  **控制实验条件**：为公平比较，人类参与者使用自己的账户和设备，以保留个性化搜索体验；智能体则在模拟器中运行，并配置了覆盖参与者地域和语言设置的多个合成身份。通过事前筛选，排除参与者凭先验知识可直接完成的任务，确保比较的是“搜索行为”本身。
2.  **强制应用内推理**：针对大模型可能依赖先验知识而非实际交互的问题，在智能体系统提示中加入了关键约束指令（如图2所示），明确要求其必须通过界面探索发现所有信息，不得使用外部知识。并对智能体运行结果进行人工审核，过滤违规运行，形成合规分析集。
3.  **细粒度的轨迹分析**：创新性地将行为比较深入到查询序列和状态导航层面。例如，通过分析首次查询的相似性，发现智能体与人类的查询在词汇上大体对齐；但通过更全面的导航行为分析，则揭示出根本差异：人类行为是**以内容为中心、探索性、高分支**的（如浏览专辑、查看相关推荐），而智能体行为是**以搜索为中心、低分支、线性**的（倾向于反复使用搜索框）。这证明了“任务结果和查询的对齐并不等同于行为对齐”。
4.  **框架的通用性与可复现性**：该框架不依赖定制化埋点，仅需屏幕录像即可重建交互轨迹，使其易于适配到其他GUI应用中进行类似的评估。

总之，论文通过这一多维度、细粒度的轨迹级评估框架，系统地诊断了人类与GUI智能体在交互策略上的深层差异，为在生产系统中将智能体作为用户代理进行部署和优化提供了关键的诊断工具和见解。

### Q4: 论文做了哪些实验？

论文在真实音频流媒体搜索应用中进行了受控实验，比较了人类用户与GUI智能体的行为。实验设置方面，39名参与者（分为专家和普通用户）与一个先进的GUI智能体（基于LLM驱动）分别执行十项多跳搜索任务。数据集基于生产环境的应用，任务设计涵盖不同熟悉度的主题。

实验主要从三个层面进行对比：任务结果与努力程度、查询表述、以及界面状态间的导航模式。对比方法包括将智能体与整体参与者、以及参与者内部子群（专家/普通、熟悉/不熟悉任务）进行统计比较。

主要结果如下：在任务成功率上，智能体（56.4%）与参与者（53.3%）相当。但智能体耗时显著更长（平均1076秒 vs. 209秒），动作数更少（中位数11 vs. 18）。查询相似性分析显示，智能体初始查询与参与者查询的相似度（SequenceMatcher相似度0.58）与参与者之间的相似度（0.57）接近，但智能体自身查询一致性更高（0.75）。分布分析表明，智能体查询覆盖了参与者查询空间的较大部分（在相似度阈值τ=0.6时覆盖46%的查询量，优于随机子集的37%）。导航行为揭示关键差异：智能体呈现搜索中心化、低分支的策略（如搜索节点自循环概率48%），而参与者则更内容中心化、探索性强（从播放列表节点更均匀地分支到播放器、艺术家等）。高频转移路径有重叠（top-10边重叠率约0.54），但低频任务特定路径差异明显。这些结果表明，尽管任务结果和查询层面大致对齐，但导航行为存在系统性不同，凸显了轨迹级评估的必要性。

### Q5: 有什么可以进一步探索的点？

该论文的局限性主要在于研究范围较窄，仅在一个特定的音频流媒体搜索应用中使用单一GUI智能体配置进行验证。虽然提出的追踪级评估框架本身是模型无关的，但所观察到的具体行为模式（如人类倾向于内容中心式探索，而智能体更偏向搜索中心式、低分支路径）可能因应用领域、界面设计或智能体架构的不同而产生差异。因此，未来研究可首先在多个生产系统（如电商、学术数据库）和多样化的智能体模型（如基于不同LLM或交互策略的Agent）中进行复现，以检验框架的普适性。

进一步探索点可围绕行为对齐的深度优化展开：一是开发能够模拟人类“探索性”与“内容中心”导航策略的智能体训练方法，例如引入强化学习中的内在好奇心机制或模仿学习，使智能体在保证任务成功的同时，生成更贴近人类决策轨迹的行为序列；二是将追踪级分析扩展到更复杂的多模态交互场景（如融合语音、手势的GUI操作），设计细粒度的跨模态行为相似性度量指标；三是构建动态演化的基准测试环境，既能反映生产系统的实时变化，又能保留关键交互状态的可复现性，以平衡真实性与实验可控性。这些方向有助于深化对智能体“拟人化”交互本质的理解，提升其在评估与优化中的代理有效性。

### Q6: 总结一下论文的主要内容

该论文提出了一种用于比较人类与GUI智能体在搜索系统中交互行为的轨迹级评估框架。研究指出，当前对GUI智能体的评估多关注任务成功率，但缺乏对其是否以类人方式交互的深入分析。为此，作者构建了一个从任务结果与努力程度、查询表述模式以及界面状态间导航行为三个维度进行对比的框架，并在一个生产级音频流搜索应用中开展了对照实验，让39名参与者和一个先进GUI智能体执行十项多跳搜索任务。

核心贡献在于揭示了任务结果和查询的对齐并不等同于行为对齐：虽然智能体取得了与人类相当的任务成功率，并生成了总体相似的查询，但其导航策略存在系统性差异。人类表现出以内容为中心、探索性的行为模式，而智能体则更倾向于以搜索为中心、低分支的路径。这一发现表明，仅基于成功的评估会掩盖行为层面的关键区别，因此轨迹级分析对于在生产搜索系统中将GUI智能体作为用户代理进行部署至关重要。该框架为诊断行为差异提供了实用方法，并启发了未来研究方向，如将类人探索策略融入智能体架构、开发超越任务成功的更丰富行为指标基准，以及设计平衡正确性与人类交互保真度的训练目标。
