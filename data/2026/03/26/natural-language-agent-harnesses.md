# Natural-Language Agent Harnesses

## 原始摘要

Agent performance increasingly depends on \emph{harness engineering}, yet harness design is usually buried in controller code and runtime-specific conventions, making it hard to transfer, compare, and study as a scientific object. We ask whether the high-level control logic of an agent harness can instead be externalized as a portable executable artifact. We introduce \textbf{Natural-Language Agent Harnesses} (NLAHs), which express harness behavior in editable natural language, and \textbf{Intelligent Harness Runtime} (IHR), a shared runtime that executes these harnesses through explicit contracts, durable artifacts, and lightweight adapters. Across coding and computer-use benchmarks, we conduct controlled evaluations of operational viability, module ablation, and code-to-text harness migration.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文试图解决的核心问题是：当前智能体（Agent）的性能越来越依赖于“控制框架工程”（harness engineering），但现有的控制框架设计通常深嵌在控制器代码和运行时特定的惯例中，缺乏可移植性、可比较性和可研究性。研究背景在于，现代智能体的成功往往取决于其外部的控制框架，该框架负责管理多步推理、工具使用、记忆、任务委派和终止等关键逻辑。尽管已有大量研究关注外部化的控制模式（如“推理-行动”循环、检索增强生成、显式自我反馈等），但现有方法存在明显不足：控制框架的逻辑通常分散在控制器代码、隐藏的框架默认设置、工具适配器、验证脚本和运行时假设中，导致框架难以在不同运行时之间迁移，难以进行公平比较，也难以进行清晰的模块化消融实验。这使得评估往往沦为对整个控制器包的比较，而非针对具体设计模块的证据分析。

因此，本文旨在将智能体控制框架的高层控制逻辑外部化为一种可移植、可执行的独立构件。具体而言，论文提出了“自然语言智能体控制框架”（NLAHs），用可编辑的自然语言来表达框架行为，并设计了“智能框架运行时”（IHR），一个通过显式契约、持久化构件和轻量级适配器来执行这些框架的共享运行时。通过这种方式，论文希望将控制框架从隐晦的代码实现提升为显式的、可执行的自然语言对象，从而促进其在不同系统间的转移、比较和科学研究。

### Q2: 有哪些相关研究？

相关研究主要可分为以下几类：

**一、提示作为程序与LLM编程系统**：如LMQL、DSPy、APPL、SGLang等工作，将提示和LLM调用视为可编程对象，关注提示的约束、控制流、管道编译或与代码的集成。本文与这些研究的区别在于，其聚焦于**治理多步智能体调用、工件契约、委派、验证和持久状态**的“**智能体挽具层**”，而非单个调用或管道。

**二、智能体控制模式与编排**：包括经典的“思考-行动”循环、检索增强、反思/自我反馈，以及后续扩展的记忆、自进化、多智能体、工作流生成和动态拓扑等模式。本文不提出新的编排算法，而是**将挽具模式逻辑外部化为共享运行时下的可执行表示**，与之形成互补。

**三、自然语言到工作流、约束与执行**：如AutoFlow、FlowAgent、Agint、AgentSpec、ContextCov、OpenProse和Lobster等系统，致力于将自然语言描述编译或转换为可执行的工作流或约束。本文的**智能挽具运行时（IHR）** 与之不同，它**直接解释挽具逻辑**，依赖显式契约和持久化工件以实现可审计性，而非编译为运行时拥有的中间表示。

**四、实践中的挽具工程与自动挽具合成**：包括处理长交互历史的上下文折叠技术、公开工程报告中强调的挽具工程重要性，以及AutoHarness、General Modular Harness等研究，后者将挽具合成视为优化目标或研究模块化结构。本文的侧重点在于**将挽具设计模式层作为一种自然语言表示对象**，并使其能在共享智能运行时下执行。

**五、可复用的指令载体与技能**：如AGENTS.md、AgentSkills等技能包，将可操作知识打包为文本以跨环境复用；相关研究还探索了技能的创建、演化、维护及评估。本文**将这一思想从可复用的局部指导，扩展到了可执行的、挽具层级的控制逻辑**。

### Q3: 论文如何解决这个问题？

论文通过引入**自然语言智能体约束**和**智能约束运行时**来解决智能体约束工程难以移植、比较和研究的问题。其核心方法是**将高层控制逻辑从代码中解耦，以可编辑的自然语言形式表达，并通过一个共享运行时来执行**。

**整体框架与架构设计**：
系统由两部分构成：1) **自然语言智能体约束**：这是一种用结构化自然语言编写的、可编辑的约束规范，定义了任务的控制流、契约和状态管理。2) **智能约束运行时**：一个共享的执行环境，负责解释并执行NLAH。IHR内部进一步分解为三个关键组件：**循环内LLM**（负责解读NLAH逻辑和当前状态，选择合规的下一步动作）、**后端**（提供工具接口和多智能体管理功能，如生成子智能体、处理返回产物）以及**运行时章程**（明确定义了契约、状态、编排和子智能体生命周期的语义）。

**主要模块/组件与关键技术**：
1.  **显式契约**：将单次模型调用提升为受明确执行契约约束的智能体调用，规定了必需输出、预算、权限、完成条件和指定输出路径。
2.  **结构化NLAH组件**：NLAH明确包含了几个核心部分：**契约**（输入输出、格式约束、验证门控、重试规则）、**角色**（具有不重叠职责的角色提示，如求解器、验证器）、**阶段结构**（明确的工作流拓扑，如计划→执行→验证→修复）、**适配器与脚本**（用于确定性操作的命名钩子，如测试、验证器）、**状态语义**（定义跨步骤持久化的内容及如何重新打开）以及**故障分类**（驱动恢复的命名故障模式）。
3.  **文件支持的状态模块**（创新点）：为了解决长程自主性中状态隐式或易失的问题，论文研究了一个可选模块，将持久状态外部化为**路径可寻址的产物**。该模块强制实现三个属性：**外部化**（状态写入产物而非仅保存在临时上下文中）、**路径可寻址**（后续阶段通过路径精确重开对象）以及**压缩稳定**（状态在截断、重启和委托后依然存在）。这提升了在上下文截断和分支情况下的稳定性。
4.  **清晰的边界划分**：方法论上明确区分了**约束**（承载任务族策略，如阶段、产物契约、验证器）和**运行时**（提供通用服务，如工具适配器、沙盒、子智能体生命周期管理），以便于在共享运行时假设下对约束模式逻辑进行比较、迁移和消融研究。

**创新点**：
主要创新在于提出了**用自然语言作为便携、可执行、可编辑的约束规范**这一思想，并通过**智能约束运行时**（特别是其循环内LLM解释器、显式契约和文件支持的状态管理）来实现其执行。这使得约束设计从埋藏在控制器代码中的实现细节，转变为一个独立的、可移植、可比较和可研究的科学对象。

### Q4: 论文做了哪些实验？

论文在编码和计算机使用两个基准测试家族上进行了三组对照实验，以评估自然语言智能体框架（NLAH）和智能框架运行时（IHR）的可行性。

**实验设置与数据集**：实验使用统一的IHR实例化，后端为Codex CLI，模型为GPT-5.4，推理强度设为“xhigh”。所有实验在Docker容器中运行，资源受限。由于预算限制，实验在固定随机种子下对基准测试子集进行采样，包括125个SWE-bench Verified样本（评估代码仓库问题解决率）和36个OSWorld样本（评估桌面环境任务成功率）。

**对比方法与主要结果**：
1.  **RQ1（行为影响）**：在固定预算下，比较完整IHR与去除运行时技能（w/o RTS）或去除框架技能（w/o HS）的消融设置。主要发现是，完整IHR显著改变了过程指标（如提示词数量、工具调用次数），但任务解决率变化不大。例如，在SWE-bench上，TRAE框架的完整IHR设置性能为74.4%，但提示词达1630万，工具调用642.6次；而w/o RTS设置性能为76.0%，提示词仅1110万，工具调用451.9次。这表明共享运行时和框架逻辑是真实的行为控制器，而非简单的提示包装。
2.  **RQ2（可组合性）**：从基准特定的基础起点开始，逐一添加模块（如文件支持状态、证据支持回答、验证器、自我进化、多候选搜索、动态编排），以研究模块级组合与消融。结果显示，模块效果集中在少数边界案例上，而非均匀提升整体性能。例如，在SWE-bench上，“自我进化”模块将性能从75.2%提升至80.0%（+4.8%），而“多候选搜索”模块则降至72.8%（-2.4%）。这表明模块的有效性取决于其能否使中间行为与评估者的最终接受条件更紧密对齐。
3.  **RQ3（迁移）**：将原生代码实现的框架与在IHR下重建的NLAH版本进行配对比较。以OSWorld上的OS-Symphony框架为例，原生代码实现的成功率为30.4%，而NLAH版本达到47.2%。关键差异在于行为模式：NLAH版本更倾向于围绕文件支持的状态和工件支持的验证来组织任务，将可靠性机制从脆弱的GUI屏幕修复转向持久的运行时状态，从而获得了适度的性能提升。

### Q5: 有什么可以进一步探索的点？

该论文提出的自然语言智能体“马具”（NLAH）框架，其核心局限在于自然语言固有的模糊性，难以精确复现依赖隐藏状态、专有调度器或训练诱导行为的复杂控制逻辑。同时，共享运行时（IHR）可能“吸收”部分本应归属于马具文本的行为，导致归因模糊，模块消融实验也难以严格确定因果关系。

未来研究可从以下几个方向深入：首先，**增强形式化与混合表达**，探索将关键控制逻辑（如条件分支、循环）与自然语言描述结合，或开发领域特定语言（DSL）来平衡可读性与精确性。其次，**提升鲁棒性与安全性**，需设计更严格的合约验证、沙箱隔离与供应链审计机制，以防范由可移植工作流带来的新型攻击面（如提示注入、恶意工具嫁接）。最后，**深化科学评估方法**，需要开发更严谨的实验框架来量化自然语言描述与代码实现之间的行为差异，并建立标准化的马具语料库，以促进其作为“科学对象”的比较、复用与演进研究。

### Q6: 总结一下论文的主要内容

该论文探讨了如何将智能体性能依赖的“控制逻辑层”（harness）从底层代码中解耦，使其成为可移植、可比较、可研究的科学对象。核心贡献是提出了**自然语言智能体控制逻辑**（NLAHs），用可编辑的自然语言描述控制逻辑行为，并设计了**智能控制逻辑运行时**（IHR），通过明确的合约、持久化构件和轻量适配器来执行这些逻辑。

论文的问题在于传统控制逻辑设计深嵌于控制器代码和运行时特定约定中，难以移植、比较和研究。方法上，作者通过NLAHs将高层控制逻辑外部化为可执行的文本描述，并由IHR提供统一的运行时解释。在编码和计算机使用基准测试中，论文通过控制实验验证了该方案的操作可行性、模块化组合与消融能力，以及从代码到文本的控制逻辑迁移研究的有效性。

主要结论表明，该框架为“控制逻辑表示科学”开辟了路径，使控制逻辑模块成为一等研究构件，而非围绕模型的偶然性粘合代码，从而促进了智能体系统的可移植性、可比较性和可研究性。
