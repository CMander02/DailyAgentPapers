# MemCollab: Cross-Agent Memory Collaboration via Contrastive Trajectory Distillation

## 原始摘要

Large language model (LLM)-based agents rely on memory mechanisms to reuse knowledge from past problem-solving experiences. Existing approaches typically construct memory in a per-agent manner, tightly coupling stored knowledge to a single model's reasoning style. In modern deployments with heterogeneous agents, a natural question arises: can a single memory system be shared across different models? We found that naively transferring memory between agents often degrades performance, as such memory entangles task-relevant knowledge with agent-specific biases. To address this challenge, we propose MemCollab, a collaborative memory framework that constructs agent-agnostic memory by contrasting reasoning trajectories generated by different agents on the same task. This contrastive process distills abstract reasoning constraints that capture shared task-level invariants while suppressing agent-specific artifacts. We further introduce a task-aware retrieval mechanism that conditions memory access on task category, ensuring that only relevant constraints are used at inference time. Experiments on mathematical reasoning and code generation benchmarks demonstrate that MemCollab consistently improves both accuracy and inference-time efficiency across diverse agents, including cross-modal-family settings. Our results show that the collaboratively constructed memory can function as a shared reasoning resource for diverse LLM-based agents.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决大型语言模型（LLM）智能体在异构部署环境中，其记忆系统难以跨模型共享和复用的问题。研究背景是，当前基于LLM的智能体常依赖记忆机制来存储和复用过往的问题解决经验，以提升推理能力和效率。然而，现有方法（如BoT）通常以“单智能体”为中心构建记忆，即记忆源自并服务于同一个模型或智能体自身的推理轨迹。这种做法将任务相关知识与其特定模型的推理风格、偏好和偏差紧密耦合，导致记忆内容高度“个性化”。

现有方法的不足在于，随着实际应用越来越多地涉及由不同规模、架构或角色的模型组成的异构智能体生态系统，这种“一对一”的紧耦合记忆范式变得低效且不适用。论文通过实验发现，若简单地将一个智能体的记忆直接迁移给另一个智能体使用，往往会因为记忆中混杂了源智能体的特定偏差（如偏好的解题策略、习惯性启发式方法），而非纯粹的、可迁移的任务知识，从而导致性能下降。

因此，本文要解决的核心问题是：**能否构建一个单一的、与具体智能体无关（agent-agnostic）的记忆系统，使其能够作为一种共享的推理资源，有效且普适地服务于多样化的LLM智能体？** 为了解决这个问题，论文提出了MemCollab框架，其核心思想是通过对比不同智能体在同一任务上产生的推理轨迹，进行对比性轨迹蒸馏，从而提炼出捕捉共享任务级不变性（如通用的推理约束、错误规避模式）的抽象知识，同时抑制智能体特有的“噪声”和偏差。此外，论文还引入了任务感知的检索机制，确保在推理时只检索与当前任务类别相关的记忆约束，以提升记忆使用的精准性和效率。

### Q2: 有哪些相关研究？

相关研究主要可分为方法类和应用类。在方法类中，现有记忆机制包括：记录交互的**情景记忆**，支持持续回忆与更新的**长期交互记忆**，以及提炼可复用指导的**推理导向记忆**。这些方法通常聚焦于提升单个智能体，记忆内容源自并服务于同一智能体，限制了跨智能体复用。此外，研究还涉及记忆的组织与操作，如基于图结构的记忆链接演化，以及通过强化学习训练的记忆管理策略。在应用类中，**检索增强生成（RAG）** 通过检索外部语料库来增强LLM的事实性与覆盖度，检索源已从非结构化文本扩展到半结构化、结构化数据及LLM自身生成内容，检索粒度也从文档级细化到句子、实体或代码片段。

本文与这些工作的关系和区别在于：不同于上述单智能体记忆或通用知识检索，本文提出**跨智能体记忆协作框架**，旨在构建一个可服务于不同模型（包括异构架构）的共享记忆系统。具体而言，本文通过对比不同智能体在同一任务上的推理轨迹，提炼出与智能体无关的抽象推理约束和错误禁止模式，从而抑制智能体特定偏差，实现任务级不变知识的迁移。在检索机制上，本文采用任务感知的粗粒度检索，先进行任务分类以缩小搜索范围，避免了细粒度检索可能带来的噪声与开销，专注于检索与任务相关的推理指导，而非外部知识文档。

### Q3: 论文如何解决这个问题？

论文通过提出MemCollab框架来解决跨智能体共享记忆的挑战，其核心方法是**对比轨迹蒸馏**，旨在从异构智能体的推理轨迹中提炼出与智能体无关的、可重用的高层推理策略和约束。

**整体框架与主要模块**：
MemCollab框架分为两个核心阶段：
1.  **对比驱动的记忆构建**：首先，对于训练集中的每个任务，让一对异构智能体（如不同规模或架构的模型）独立生成推理轨迹。通过一个指示模型评估轨迹的正确性，确定“优选”和“非优选”轨迹。然后，利用一个基础大语言模型对比分析这两条轨迹，总结它们在结构推理层面的差异。这个过程会蒸馏出两种抽象知识：**违反模式**（即非优选轨迹中系统性的推理错误，如过早计算）和**推理不变量**（即优选轨迹中保持的、避免错误的结构性原则）。这些知识被形式化为“执行某不变量；避免某模式”的规范约束，并存入记忆库。记忆库存储的是抽象的、规范化的推理约束，而非原始轨迹或具体答案。
2.  **任务感知的记忆检索与推理**：在推理时，面对新任务查询，首先使用基础大语言模型对查询进行**任务分类**（确定任务类别和子类别）。记忆库中的每条约束在构建时都已标记其来源任务的类别。检索时，系统根据查询的任务类别进行过滤，仅从匹配类别的约束中检索最相关的若干条。最后，智能体在生成解决方案时，以这些检索到的规范约束作为软性指导，引导其遵循正确的推理结构，避免已知的错误模式。

**架构设计与关键技术**：
*   **对比轨迹分析**：这是核心创新点。通过对比同一任务下成功与失败的轨迹，直接聚焦于区分正确与错误推理的关键结构性因素，从而剥离了智能体特有的风格偏好或启发式捷径等偏差，提炼出任务层面共享的、可迁移的推理约束。
*   **任务感知检索机制**：为了解决无关记忆引入噪声的问题，创新性地将记忆库按任务类别进行组织。在检索前先对查询任务分类，然后进行基于类别的过滤和排序。这比单纯基于相似度的检索更能确保相关性和减少干扰。
*   **规范约束表示**：记忆条目使用抽象的推理概念（如累加、条件判断、比例关系、枚举案例等）来表达，而非具体任务细节，这增强了记忆在不同问题实例间的泛化能力。

**创新点**：
1.  **问题定义创新**：首次系统性地探索了为异构大语言模型智能体构建**共享记忆系统**的可能性与挑战。
2.  **方法创新**：提出了**对比轨迹蒸馏**作为构建智能体无关记忆的核心技术，通过对比分析来分离任务不变性与智能体特异性偏差。
3.  **系统设计创新**：引入了**任务感知的层次化记忆组织与检索机制**，有效提升了记忆检索的精度和推理效率，避免了无关约束的干扰。

### Q4: 论文做了哪些实验？

论文在数学推理和代码生成任务上进行了全面的实验评估。实验设置方面，使用了Qwen2.5-7B-Instruct、Qwen2.5-32B-Instruct和LLaMA-3-8B-Instruct作为骨干模型。数据集包括数学推理的MATH500和GSM8K（各使用1000个实例构建记忆系统，在500个独立实例上评估），以及代码生成的MBPP和HumanEval，以准确率（%）作为核心指标。

对比方法分为两类：一是记忆内容基线，包括Buffer of Thoughts (BoT)和Dynamic Cheatsheet (DC)；二是记忆来源基线，包括使用单一模型轨迹总结的记忆（如w/ Memory(32B)）和基于同一模型多轨迹对比构建的自对比记忆（Self-Contrast Memory）。

主要结果显示，MemCollab在多个基准上显著提升了不同智能体的性能。关键数据指标如下：对于Qwen-2.5-7B，MemCollab在MATH500上准确率从52.2%提升至67.0%，MBPP从47.9%提升至57.6%，平均准确率达到71.6%，超越了所有基线。对于Qwen-2.5-32B，MemCollab在MATH500达到73.8%，GSM8K达到93.6%，平均79.6%，同样表现最佳。在跨模型架构（如LLaMA-3-8B）实验中，MemCollab也带来了显著提升，例如LLaMA-3-8B在MATH500上从27.4%提升至42.4%。此外，实验还表明MemCollab提高了推理效率，例如Qwen2.5-7B在MBPP上的平均推理轮数从3.1降至1.4。消融研究确定了检索3个记忆条目为最优，并验证了基于任务类别的检索机制优于基于嵌入相似度或模型直接选择的方法。

### Q5: 有什么可以进一步探索的点？

本文提出的跨智能体记忆协作框架虽具创新性，但仍存在若干局限与可拓展方向。首先，其对比轨迹蒸馏方法依赖于多个智能体对同一任务生成轨迹，这在现实场景中可能成本较高，且若智能体多样性不足，可能无法充分提炼通用约束。未来可探索如何利用少量高质量轨迹或合成数据来构建记忆，降低依赖。其次，当前框架主要针对数学推理和代码生成等结构化任务，未来可验证其在开放域对话、复杂决策等更模糊任务上的泛化能力。此外，论文提及未来可将对比形式扩展至多智能体环境，通过选择偏好轨迹构建共享记忆系统，这一方向值得深入，例如引入强化学习或博弈论方法动态优化轨迹选择策略。最后，记忆检索机制目前仅基于任务类别，未来可融入更细粒度的语义相似度计算，实现更精准的记忆匹配，从而进一步提升跨模型协作效率。

### Q6: 总结一下论文的主要内容

该论文提出了MemCollab框架，旨在解决异构大语言模型（LLM）智能体之间共享记忆的挑战。现有方法通常为每个智能体单独构建记忆，导致记忆与特定模型的推理风格紧密耦合，难以在不同模型间有效复用。论文发现，简单地跨智能体转移记忆会因记忆混杂了任务相关知识和智能体特定偏差而导致性能下降。

为解决此问题，MemCollab的核心方法是**对比轨迹蒸馏**。它通过对比不同智能体在同一任务上生成的推理轨迹，提炼出抽象的、与智能体无关的推理约束。这些约束捕捉了任务层面的共享不变性，同时抑制了智能体特有的“痕迹”。此外，框架引入了**任务感知检索机制**，根据任务类别来访问记忆，确保在推理时只使用相关的约束。

实验在数学推理和代码生成基准上进行。结果表明，MemCollab能持续提升不同智能体（包括跨模型家族设置）的准确性和推理效率。其主要结论是，通过对比蒸馏构建的协作记忆可以作为多样化LLM智能体共享的推理资源，实现跨智能体的高效知识复用，这是其核心贡献与意义所在。
