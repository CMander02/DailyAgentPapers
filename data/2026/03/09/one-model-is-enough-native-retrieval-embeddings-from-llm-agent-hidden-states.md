# One Model Is Enough: Native Retrieval Embeddings from LLM Agent Hidden States

## 原始摘要

LLM agents that retrieve external knowledge typically generate a search query as text, then run a separate embedding model to encode it into a vector. This two-model pipeline adds infrastructure complexity and latency, yet is redundant: the LLM already encodes the full conversational context in its hidden states. We propose equipping LLM agents with native retrieval capability by adding a lightweight projection head that maps hidden states directly into the embedding space, eliminating the need for a separate embedding model. Trained with a combination of alignment, contrastive, and rank distillation losses, our method retains 97\% of baseline retrieval quality while enabling the LLM agent to search with its own representations. Experiments on the QReCC conversational search benchmark show competitive Recall@10 and MRR@10 compared to the standard generate-then-encode pipeline, with systematic ablations confirming the contribution of each loss component.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决基于检索增强生成（RAG）的大语言模型（LLM）智能体在知识检索时存在的架构冗余和效率问题。当前，当LLM智能体需要搜索外部知识时，标准流程采用“先生成后编码”的两阶段管道：LLM首先生成一个自然语言查询文本，随后一个独立的专用嵌入模型将该文本编码为向量以进行检索。这种主流范式虽然有效，但存在明显不足。研究背景是，LLM在处理完整对话上下文（包括用户意图、历史记录和任务需求）时，其内部隐藏状态已经蕴含了丰富的语义理解，而生成的查询文本只是这种内部表征的一种有损的、离散的投影。现有方法的根本缺陷在于，独立的嵌入模型需要重新处理这个文本，近似地恢复LLM已经掌握的信息，这造成了计算和语义上的冗余，并增加了系统复杂性和延迟。

因此，本文要解决的核心问题是：能否消除对独立嵌入模型的依赖，让LLM智能体具备“原生检索”能力？论文提出，既然LLM的隐藏状态已包含检索所需信息，只需通过一个轻量级的投影头，将这些隐藏状态直接映射到现有的嵌入向量空间即可。这样，智能体就能使用自身的内部表征进行搜索，从而简化架构、降低延迟，同时保持高质量的检索效果。

### Q2: 有哪些相关研究？

本文的相关研究主要可分为以下几类：

**检索增强生成（RAG）与对话式检索**：这是本文工作的直接背景。标准RAG系统采用“检索-生成”流水线，依赖一个独立的专用嵌入模型来编码查询。对话式RAG则需处理多轮对话中的指代消解和话题转移。本文质疑并旨在消除对专用嵌入模型的依赖，直接利用LLM自身隐藏状态进行检索。

**对话查询改写**：为解决多轮检索中的歧义性，这类方法使用LLM将含对话历史的原始查询改写成自包含的搜索查询文本。然而，生成的文本仍需由独立的嵌入模型编码。本文方法则完全绕过了这一编码步骤，直接从已包含完整上下文的LLM隐藏状态进行投影。

**基于LLM的嵌入模型**：如LLM2Vec和NV-Embed等工作探索将预训练LLM改造为通用文本嵌入模型。它们通常需要修改LLM架构（如双向注意力）并进行专门训练，在嵌入模式下仍需一次独立的前向传播。本文与之关键区别在于：**不将LLM作为独立的嵌入模型运行**，而是仅通过一个轻量级投影头，利用LLM正常生成推理过程中**已计算出的**隐藏状态来产生嵌入，避免了额外的完整前向传播开销。

**检索中的知识蒸馏**：知识蒸馏（包括专注于排序传递的Rank Distillation）常用于压缩检索模型，训练一个更小的学生编码器来模仿教师模型。本文场景不同：学生模型并非独立编码器，而是附加在LLM上的投影头；目标不是模型压缩，而是完全消除独立的编码器。

**假设文档嵌入（HyDE）**：HyDE方法为查询生成一个假设文档，然后嵌入该文档以改进检索匹配。这种方法增加了生成成本且仍需要嵌入模型。本文则反其道而行之：**避免为检索目的生成任何额外文本**，直接从现有上下文中获取嵌入。

### Q3: 论文如何解决这个问题？

论文通过为LLM智能体增加一个轻量级的投影头，使其能够直接利用自身生成过程中产生的隐藏状态来生成检索向量，从而消除对独立嵌入模型的依赖。核心方法是将LLM在生成搜索查询文本时自然计算出的隐藏状态序列，映射到目标嵌入空间。

整体框架与主要模块如下：首先，在LLM以自回归方式生成查询文本时，同步收集每个生成token对应的最后一层隐藏状态，形成一个隐藏状态序列。这一过程几乎不增加额外开销，因为这些状态本就是生成过程的副产品。随后，一个专门设计的投影头对该序列进行处理。该投影头由四个组件构成：1）输入投影层，将LLM的隐藏维度线性映射到投影头内部维度；2）一个多层Transformer编码器，配备可学习的位置编码，用于聚合整个序列的信息并捕捉token间的依赖关系；3）池化层，对编码后的序列进行平均池化，得到一个单一的向量表示；4）输出投影与归一化层，将池化后的向量投影到目标嵌入维度，并进行L2归一化，以确保点积检索等价于余弦相似度计算。

关键技术在于通过知识蒸馏训练这个投影头。训练数据是查询的隐藏状态序列与独立嵌入模型（教师模型）生成的对应嵌入向量对。训练采用了三种损失的组合：1）对齐损失，直接最小化投影输出与教师嵌入之间的角度距离；2）对比损失，采用InfoNCE损失，鼓励投影嵌入在批次内保持查询间的判别性结构；3）排序蒸馏损失，通过KL散度将教师模型对候选文档的排序偏好迁移给学生模型。最终损失是这三个损失的加权和。

该方法的创新点在于：首次提出并实现了LLM智能体的“原生检索”能力，利用其自身已有的计算产物（隐藏状态）来替代独立的检索编码模型，从而简化了架构、降低了延迟和基础设施复杂性。实验表明，该方法在保持97%基线检索质量的同时，实现了端到端的检索流程。

### Q4: 论文做了哪些实验？

论文在QReCC对话式搜索基准上进行了实验，这是一个包含多轮对话、人工改写查询和相关文档标注的大规模数据集。实验使用了qrecc_eval_large分割，包含346个对话和2189个检索触发点。实验设置采用同系列模型：以Qwen3-8B作为LLM智能体，Qwen3-Embedding-8B作为教师嵌入模型。基线方法是标准的“生成-再编码”流程：LLM生成改写后的搜索查询，再由独立的嵌入模型编码为向量。提出的方法则是在LLM的隐藏状态（维度4096）上添加一个轻量级投影头（约2500万参数），直接映射到目标嵌入空间（维度1024），从而省去单独的嵌入模型。

主要对比了提出的方法与基线在检索质量（Recall@10、MRR@10、nDCG@10）和推理延迟（p50）上的表现。关键结果显示，提出的方法在保持接近基线检索质量的同时，大幅降低了延迟：Recall@10为0.607（基线0.637，保留95.1%），MRR@10为0.293（基线0.329，保留89.1%），nDCG@10为0.367（基线0.402，保留91.3%）。延迟从43.5ms降至2.0ms，加速了21.8倍。统计检验（McNemar's test）表明质量差距显著（p=0.0005）。

论文还进行了系统的消融实验，以验证各损失组件的作用。实验表明，对齐损失（alignment loss）单独使用效果最强（Recall@10 0.567），对比损失（contrastive loss）单独效果次之（0.498），而排序蒸馏损失（rank distillation loss）单独使用会完全失效（0.001）。三者结合（权重各0.5）在标准训练下达到Recall@10 0.595，配合延长训练周期（80轮）和更低学习率（2e-4）后提升至0.607。此外，实验还探讨了训练周期、学习率、排序蒸馏权重和候选文档数量K的影响，发现学习率过高（5e-4）会导致训练崩溃，而适度增加K（如128到256）影响不大。

### Q5: 有什么可以进一步探索的点？

该论文的局限性为未来研究提供了多个明确方向。首先，方法仅在单一数据集（QReCC）上验证，其泛化能力有待在更广泛的对话搜索基准、开放域检索乃至多模态任务中进行系统性评估。其次，实验局限于同系列模型（Qwen3），未来需探索跨模型家族（如从Llama映射到BGE嵌入空间）的投影能力，这对方法的通用性至关重要。第三，虽然性能差距小但统计显著，如何通过改进损失函数设计（如引入更精细的排序学习目标）或架构优化来完全追平基线，是核心挑战。此外，训练仍依赖外部嵌入模型生成数据，未来可研究自监督或蒸馏方法以摆脱此依赖。最后，投影头的分布外泛化能力、动态上下文长度适应性，以及将其集成到更复杂智能体工作流（如与工具调用协同）中的潜力，都是值得深入探索的课题。

### Q6: 总结一下论文的主要内容

这篇论文针对LLM智能体在检索外部知识时需要先生成文本查询、再调用独立嵌入模型进行编码的低效流程，提出了一个创新解决方案。核心问题是现有两阶段管道导致基础设施复杂、延迟高且存在冗余，因为LLM本身的隐藏状态已包含完整对话上下文信息。为此，作者提出为LLM智能体赋予原生检索能力，方法是在LLM上添加一个轻量级投影头，直接将隐藏状态映射到嵌入空间，从而省去独立的嵌入模型。该方法结合了对齐损失、对比损失和排序蒸馏损失进行训练，在QReCC对话式搜索基准上的实验表明，其检索质量达到了基线方法的97%，且在Recall@10和MRR@10指标上与传统生成-编码流程表现相当。主要结论是，仅用一个LLM模型即可实现高效检索，显著简化了系统架构并降低了延迟，同时通过消融实验验证了各损失成分的有效性。这一工作为构建更简洁、高效的检索增强型LLM智能体提供了重要思路。
