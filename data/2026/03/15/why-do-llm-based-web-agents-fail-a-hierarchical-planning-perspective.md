# Why Do LLM-based Web Agents Fail? A Hierarchical Planning Perspective

## 原始摘要

Large language model (LLM) web agents are increasingly used for web navigation but remain far from human reliability on realistic, long-horizon tasks. Existing evaluations focus primarily on end-to-end success, offering limited insight into where failures arise. We propose a hierarchical planning framework to analyze web agents across three layers (i.e., high-level planning, low-level execution, and replanning), enabling process-based evaluation of reasoning, grounding, and recovery. Our experiments show that structured Planning Domain Definition Language (PDDL) plans produce more concise and goal-directed strategies than natural language (NL) plans, but low-level execution remains the dominant bottleneck. These results indicate that improving perceptual grounding and adaptive control, not only high-level reasoning, is critical for achieving human-level reliability. This hierarchical perspective provides a principled foundation for diagnosing and advancing LLM web agents.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决当前基于大语言模型（LLM）的网页智能体在现实、长周期任务中可靠性远低于人类的问题。研究背景是，尽管LLM网页智能体在网页浏览和多步骤在线工作流方面取得了显著进展，有望自动化日常在线交互，但现有评估方法主要依赖粗粒度的端到端成功率指标，这无法揭示失败的具体根源。现有方法的不足在于，它们无法有效诊断失败究竟源于错误的任务理解、有缺陷的高层策略、将计划具体化为UI操作时的“落地”能力差，还是环境偏离预期时缺乏有效的恢复能力。

因此，本文要解决的核心问题是：如何对LLM网页智能体的失败原因进行系统性、细粒度的诊断和分析。为此，论文提出了一个分层规划视角的评估框架，将智能体的运作分解为三个层次——高层规划、低层执行和重新规划，从而能够对模型的推理、落地和恢复能力进行基于过程的评估。通过这一框架，研究旨在识别性能瓶颈的具体所在，为有针对性地提升智能体可靠性提供原则性基础。

### Q2: 有哪些相关研究？

本文的相关研究主要可分为三大类：评测方法、高层规划和低层执行与重规划。

**1. 评测方法类**：早期评测依赖离线环境下的轨迹精确匹配，难以适应动态网页。近期研究转向人工评估、基于规则的成功标准或使用LLM作为评判员来评估轨迹或最终状态。本文与这些工作的区别在于，它提出了一个分层规划框架，将高层规划、低层执行和重规划解耦进行过程性评估，旨在深入诊断失败根源，而非仅关注端到端成功率。

**2. 高层规划类**：已有研究探索利用LLM生成高层规划，例如输出PDDL等结构化表示供经典规划器使用，或在网页领域将用户指令分解为抽象子目标。本文延伸了这条路线，首次将形式化规划表示（PDDL）引入网页智能体，并系统比较了自然语言规划与PDDL规划的效果。

**3. 低层执行与重规划类**：为提升低层执行可靠性，相关研究包括简化网页观察以对齐LLM表示、将低层交互抽象为网站特定工具以提高鲁棒性，或通过训练从轨迹直接学习动作。在重规划方面，早期智能体框架展示了在部分可观测环境中根据观察进行推理与行动调整的潜力，后续研究也利用环境反馈进行重规划。本文的贡献在于，它提出了在明确高层目标下评估低层执行的指标，并强调保持规划与执行的分离，以准确定位瓶颈。

### Q3: 论文如何解决这个问题？

论文通过提出一个分层规划框架来分析和诊断LLM网络智能体失败的原因，并基于此框架进行系统性评估。其核心方法是构建一个三层结构（高层规划、低层执行和重规划）的智能体模型，将复杂的网络导航任务分解为可独立评估的模块，从而精确定位性能瓶颈。

整体框架遵循经典的分层任务网络（HTN）思想。主要模块包括：1）**高层规划器**：接收自然语言指令，生成一系列抽象的子目标序列。论文对比了自然语言（NL）和规划域定义语言（PDDL）两种表示形式，发现结构化的PDDL规划能产生更简洁、目标更明确的策略，与人类标注计划的匹配度更高。2）**低层执行器**：负责将每个子目标实例化为具体的、可执行的网络交互动作序列（如点击、搜索）。这是框架中的关键执行组件。3）**后条件检查器**：使用LLM作为评判员，评估执行轨迹后状态是否满足子目标的预期效果，为是否进入下一子目标或触发重规划提供反馈信号。4）**重规划机制**：当子目标执行失败或陷入僵局时，智能体可进行局部调整（从最后一个成功子目标继续规划）或全局重规划（基于当前上下文生成全新的子目标序列）。

关键技术在于利用这一分层框架进行**过程性评估**，而不仅仅是端到端的成功率评估。论文设计了一系列研究问题（RQ1-RQ7）和对应的量化指标，分别审视三层能力：通过“完美匹配率”、“部分匹配率”等指标评估高层规划与人类策略的对齐程度；通过“子目标完成率”、“计划完成率”和“任务成功率”等指标评估低层执行的可靠性；通过重规划前后对齐度与成功率的变化评估恢复能力。

创新点主要体现在：1）**诊断视角的创新**：首次从分层规划的视角系统解构LLM网络智能体的失败原因，明确指出低层执行（感知 grounding 和自适应控制）是当前的主要瓶颈，而非仅高层推理。2）**评估方法的创新**：提出了一个基于分层能力的评估框架，能够将总体错误归因于特定的核心能力缺陷，避免了传统结果评估的模糊性。3）**结构化表示的验证**：通过实验证实，在高层规划中使用PDDL等结构化表示能显著改善规划质量，这为改进智能体设计提供了明确方向。

### Q4: 论文做了哪些实验？

实验基于Mind2Web-Live基准测试集的104个真实网页任务实例，使用专家标注的关键节点作为黄金标准高层子目标。主要模型为gpt-5-nano，并对比了gemini-flash-2.5和claude-haiku-4.5。实验设置包含三个层次：高层规划（比较自然语言NL与结构化PDDL两种表示）、低层执行（比较扩展动作空间、动作-对象预测和动作ID选择三种设置）以及重规划。

主要结果如下：在高层规划对齐方面，PDDL规划比NL规划更接近人类标注，其步骤匹配率达84.6%（NL为70.6%），完美匹配率67.7%（NL为60.6%），且规划更简洁（平均高层子目标数PDDL为4.39，NL为5.51，人类为3.04）。低层执行是主要瓶颈：即使给定人类规划，执行器仅达成38.5%的子目标完成率和36.4%的最终成功率。不同动作表示的错误率差异显著，动作-对象预测的幻觉率达34.0%，而动作ID选择和扩展动作空间分别仅为2.0%和3.0。常见失败模式包括冗余动作（占全部动作34.2%）、幻觉链接（占跳转动作32.0%）和重复执行（占失败10.4%）。重规划后，PDDL规划稳定性更高（匹配率保持84.6%），而NL规划匹配率从70.6%降至65.0%；但重规划显著提升了子目标完成率。模型对比显示，gpt-5-nano综合性能最优，最终成功率36.4%，高于claude-haiku-4.5（29.2%）和gemini-flash-2.5（17.3%）。

### Q5: 有什么可以进一步探索的点？

基于论文分析，未来可探索的方向主要集中在以下几个方面：首先，论文指出低层执行是当前智能体的主要瓶颈，尤其是感知对齐和状态跟踪问题。未来研究可以设计更丰富的环境表征方法，例如结合视觉或多模态信息，以提升模型对动态网页元素的准确理解和交互能力。其次，论文建议将规划与执行作为分离技能进行优化，未来可探索模块化智能体架构，其中高层规划使用结构化表示（如PDDL）生成抽象子目标，而低层控制器则专注于鲁棒的环境对齐，并引入显式的不确定性处理机制（如允许执行器请求澄清或重新规划）。此外，论文的评估方法依赖于人工标注的参考计划，限制了灵活性。未来可研究自动化或半自动化的计划标注技术，以降低评估成本并扩展框架的适用性。最后，当前研究未涉及多模态设置，未来可探索融合文本、图像和页面结构的混合表征，以更全面地模拟人类在复杂网页任务中的决策过程。

### Q6: 总结一下论文的主要内容

该论文从分层规划视角系统分析了基于大语言模型的网页智能体在复杂、长周期任务中表现不佳的原因。核心贡献在于提出了一个包含高层规划、低层执行和重规划的三层评估框架，实现了对智能体推理、感知对齐和恢复能力的全流程诊断。

研究将规划表述分为自然语言和结构化规划域定义语言两种形式，实验表明结构化规划能产生更简洁、目标导向的策略，但低层执行（如感知对齐和自适应控制）仍是主要性能瓶颈。这一发现揭示了提升网页智能体可靠性的关键不仅在于改进高层推理，更需增强其对网页环境的精准感知与动态调整能力。

该工作为系统诊断智能体失败原因提供了理论依据，并指明了未来研究应重点关注执行层面的技术突破，而非仅优化规划能力，从而推动智能体向人类水平可靠性迈进。
