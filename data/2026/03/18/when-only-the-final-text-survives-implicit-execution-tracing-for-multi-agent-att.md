# When Only the Final Text Survives: Implicit Execution Tracing for Multi-Agent Attribution

## 原始摘要

When a multi-agent system produces an incorrect or harmful answer, who is accountable if execution logs and agent identifiers are unavailable? Multi-agent language systems increasingly rely on structured interactions such as delegation and iterative refinement, yet the final output often obscures the underlying interaction topology and agent contributions. We introduce IET (Implicit Execution Tracing), a metadata-independent framework that enables token-level attribution directly from generated text and a simple mechanism for interaction topology reconstruction. During generation, agent-specific keyed signals are embedded into the token distribution, transforming the text into a self-describing execution trace detectable only with a secret key. At detection time, a transition-aware scoring method identifies agent handover points and reconstructs the interaction graph. Experiments show that IET recovers agent segments and coordination structure with high accuracy while preserving generation quality, enabling privacy-preserving auditing for multi-agent language systems.

## Q&A 论文解读

### Q1: 这篇论文试图解决什么问题？

这篇论文旨在解决多智能体语言系统在缺乏执行日志和身份标识情况下的责任归属问题。随着多智能体系统在复杂任务中的广泛应用，其高失败率（41%至87%）和操作不透明性使得当系统产生错误或有害内容时，难以追溯具体责任方。现有方法（如Who&When和FAMAS）严重依赖“白盒”环境，需要完整的执行轨迹和明确的元数据（如模型标识、执行日志）来进行故障归因。然而，在实际生产环境中，出于隐私安全和合规性考虑，这些元数据常被剥离或抽象化，导致生成的文本与原始执行环境脱钩，一旦文本离开服务器日志环境，其与执行元数据的关联便永久丢失，使得现有基于轨迹分析的工具失效。

因此，本文的核心问题是：在仅有最终输出文本、没有显式元数据或内部系统轨迹的情况下，如何恢复多智能体系统的执行路径，并将文本片段准确归因于参与的智能体。这被称为“多智能体执行追踪”问题，其关键在于从最终文本中隐式地重建交互拓扑结构和智能体贡献，为隐私保护审计提供基础，而不仅仅是故障归因。

### Q2: 有哪些相关研究？

相关研究主要可分为三类：失败归因方法、水印技术以及多智能体系统问责机制。

在**失败归因方法**方面，现有研究主要将其视为事后推断任务。例如，基于原始执行轨迹预测责任主体（who）和关键错误步骤（when）的基准测试，以及采用反事实重演、因果分析或学习追踪模型等方法在缺乏显式溯源信息时定位失败源头。这些方法依赖于对表面交互记录的回顾性分析，但难以捕捉信息流依赖关系。本文提出的IET框架与这些方法互补，它并非仅依赖事后推断，而是在生成过程中直接嵌入可追溯性，从而能更直接地恢复智能体层级的来源信息。

在**水印技术**方面，现有工作广泛研究如何识别AI生成内容并验证模型来源。方法包括在logits生成、token采样或模型训练等阶段嵌入可检测信号，但主要关注单一模型生成的文本检测，而非追踪跨多个智能体的信息流。另一类研究通过对现有文本进行基于格式、词汇、句法或生成的变换来嵌入水印，但这些方法仅针对单一文本实例，并未建模溯源信息在多个交互智能体或中间步骤间的传播。本文的创新在于将水印重新定位为系统级工具，使其能够在多智能体交互中保存并传播溯源信息，从而实现明确且可验证的归因。

在**多智能体系统问责机制**方面，现有路径可分为“推断”和“工具化”两种。推断方法在失败发生后分析缺乏显式溯源信号的执行轨迹来确定责任；工具化方法则在生成过程中直接嵌入可追溯性。以往的多智能体失败归因研究主要遵循推断范式，而水印技术虽能嵌入可识别信号，却通常为单模型场景设计。本文的工作桥接了这两种范式，通过将基于水印的溯源机制嵌入多智能体交互，使得责任能够被确定性地恢复，并实现基于设计的审计。

### Q3: 论文如何解决这个问题？

论文通过提出一个名为“隐式执行追踪”的框架来解决多智能体系统中因缺乏执行日志和标识符而难以进行责任归属的问题。其核心方法是在生成文本时，将智能体特定的密钥信号嵌入到令牌分布中，使最终文本自身携带可检测的执行痕迹，从而在事后仅凭文本和密钥就能重建交互拓扑和进行令牌级归属。

整体框架包含两个主要阶段：**嵌入阶段**和**检测/重建阶段**。在嵌入阶段，当某个智能体生成文本时，系统会使用一个基于密钥的分布调制算子对语言模型产生的原始logits向量进行调制。该调制引入了两种由智能体身份决定的密钥：一个固定的扰动方向密钥和一个依赖于上下文的词汇排列密钥。经过调制后的logits采样生成的文本，便嵌入了该智能体的统计特征信号，且不影响生成质量。

在检测阶段，核心任务是利用密钥从生成的文本中恢复归属函数和交互图。这通过几个关键技术模块实现：
1.  **局部一致性评分**：对于文本中的每个令牌位置，计算一个统计量，用于衡量观察到的令牌与每个候选智能体所诱导信号之间的对齐程度。为减少局部波动，采用滑动窗口对评分进行平均，得到每个时间点智能体的一致性分数。
2.  **竞争性变点检测**：这是识别智能体切换点的关键。算法将多智能体评分转换为一维的竞争边际信号，该信号反映了当前主导智能体与次优候选者之间的分数差距。然后，通过累积和算法平滑该序列并定位其极值点，这些极值点即被判定为智能体交接的边界。通过一个敏感性阈值来控制检测的灵敏度。
3.  **交互拓扑重建**：为了恢复智能体之间的协作结构，论文将执行路径编码为一个在线更新的二进制邻接矩阵。每当智能体A向智能体B传递信息时，就在矩阵的对应位置标记。这个矩阵状态被唯一地编码为一个路径标识符，并同样通过水印机制嵌入到后续生成的文本中。在检测时，通过从文本中解码出这些标识符序列，可以反推出完整的邻接矩阵，从而重建出交互拓扑图。

该方法的创新点在于：首先，它实现了**元数据无关**的归属，仅依赖最终文本和密钥，无需系统提供日志或标识符，适用于信息被剥离的对抗场景。其次，它同时实现了**细粒度令牌级归属**和**高层级交互拓扑重建**。最后，其基于统计水印的嵌入方式对生成质量影响极小，且通过密钥控制保证了**审计的隐私性**，只有持有密钥的审计方才能解读出执行痕迹。

### Q4: 论文做了哪些实验？

论文在两类多智能体基准上进行了实验，以评估所提出的隐式执行追踪（IET）框架。实验设置包括两个互补的场景：拓扑感知的归因和元数据丢失下的鲁棒性归因。

**实验设置与数据集**：
1.  **拓扑感知归因**：使用**多智能体交互数据集**，该数据集包含在星型、链式、树型等典型协调拓扑下的结构化多智能体对话日志，提供了真实的智能体身份和轮次边界。
2.  **元数据丢失下的鲁棒性归因**：使用**Who & When基准**，该数据集包含带标注的对话者身份和错误归因标签。为了模拟元数据丢失，对文本施加了两种扰动：**ID移除**（移除显式的智能体标识符）和**边界破坏**（打乱或合并话语边界）。

**对比方法**：
1.  **LLM基线**：使用ChatGPT-4o和DeepSeek-v3.1，直接根据文本和智能体数量K来推断发言归属。
2.  **传统分割基线**：包括随机分割（token级和unit级）、递归分割、基于词袋语义相似度的分割以及TextTiling方法。这些方法仅能预测片段边界，无法推断发言者身份。

**主要结果与关键指标**：
1.  **顺序归因准确性**：在拓扑感知实验中，IET方法在**Token级准确率（TokenAcc）** 和**交并比（IoU）** 上显著优于所有基线。关键数据指标如下：
    *   **IET方法**：TokenAcc > 94%，IoU ~ 0.93，**拓扑准确率（TopoAcc）** 达到1.00（完美恢复）。
    *   **LLM基线**：TokenAcc < 0.30，IoU在0.10–0.18之间。
    *   **最佳分割基线（Recursive）**：IoU最高仅达0.556。
2.  **元数据丢失下的鲁棒性**：在Who & When基准的扰动测试中，IET方法在恢复后的日志上进行下游错误归因任务，评估**智能体准确率（AgentAcc）** 和**步骤准确率（StepAcc）**。结果显示，在ID移除和边界破坏两种扰动下，IET方法（表中“Ours”列）的AgentAcc和StepAcc指标普遍优于或与基线方法在未扰动时的表现相当，证明了其从纯文本中恢复归因信息并保持诊断效用的鲁棒性。例如，在部分设置下，IET在ID移除后AgentAcc仍能保持在22%-27%左右，而部分基线方法则降至0%。

### Q5: 有什么可以进一步探索的点？

该论文的局限性主要在于其实验数据集的隐私处理方式。虽然作者对包含个人身份信息（PII）的对话数据进行了标准脱敏处理（如用占位符替换敏感信息），但这种处理可能不足以应对更复杂的隐私场景。例如，在医疗、金融等高度敏感领域，即使经过脱敏，文本的语义模式或上下文仍可能隐含隐私信息，而当前的隐式执行追踪（IET）框架并未专门针对这类深层隐私风险进行优化。

未来研究方向可以从以下几个维度拓展：首先，可探索IET在动态、开放式多智能体环境（如实时协作或对抗性交互）中的鲁棒性，当前实验可能局限于结构相对固定的对话数据集。其次，可研究如何将IET与差分隐私或同态加密技术结合，在保证追踪精度的同时增强端到端的隐私保护。此外，作者未讨论智能体策略性“欺骗”或干扰追踪信号的可能性，未来可针对对抗性攻击设计防御机制。最后，可扩展IET的应用场景，如将其集成到智能体训练过程中，实现实时责任追溯与性能优化联动，从而推动多智能体系统的可信发展。

### Q6: 总结一下论文的主要内容

论文针对多智能体系统中因缺乏执行日志和标识而难以追责的问题，提出了隐式执行追踪框架。核心贡献在于无需依赖元数据，仅通过生成文本来实现细粒度的智能体归因和交互拓扑重建。方法上，通过在生成过程中嵌入密钥控制的追踪信号，使文本成为自描述的执行记录，并利用转移感知评分检测智能体切换点以重构交互图。实验表明，该框架能高精度恢复智能体贡献和协作结构，且不影响生成质量。其意义在于为多智能体系统提供了一种隐私保护的审计方案，支持在复杂生态中实现可追溯的协作。
